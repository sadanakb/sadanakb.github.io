### Post 4 — Dienstag, 18. März | Claude Code Workflow

**Typ:** Text + Bild

```
Ich werde oft gefragt, wie ich mit Claude Code arbeite.

Hier mein ehrlicher Workflow:

Ich benutze Claude Code nicht, um mir Code generieren zu lassen und dann auf "Accept" zu klicken. Das funktioniert bei Demos. Nicht bei Production Software.

So sieht es tatsächlich aus:

1. Ich plane die Architektur selbst. Welche Komponenten, welche API-Routen, welche Datenmodelle.

2. Ich schreibe Claude Code einen konkreten Auftrag. Nicht "bau mir eine App", sondern "implementiere den XRechnung-Validator nach EN 16931 Schema mit diesen Edge Cases".

3. Ich reviewe jeden Output. Zeile für Zeile. Wenn ich etwas nicht verstehe, lasse ich es mir erklären, bevor es in den Code kommt.

4. Ich teste aggressiv. 170+ Tests bei RechnungsWerk sind kein Zufall. Sie sind die Konsequenz davon, dass KI-generierter Code genauso Bugs hat wie menschlicher.

5. Ich iteriere. Der erste Output ist selten der finale. Meistens braucht es 2-3 Runden.

KI-Tools machen mich schneller. Aber nicht schlampiger. Der Unterschied entscheidet, ob am Ende ein Prototyp steht oder ein Produkt.

Wie nutzt ihr KI-Tools in eurem Workflow?
```

```
Unpopular Opinion: Die meisten Leute nutzen KI-Tools falsch.

Sie tippen "bau mir eine Website" ein, bekommen ein Ergebnis und sind enttäuscht. Oder beeindruckt. Beides aus dem falschen Grund.

KI-Tools sind kein Ersatz für Denken. Sie sind ein Multiplikator.

→ Verstehst du den Output?

Wenn die Antwort auf eine dieser Fragen "nein" ist, hast du kein Produkt. Du hast einen Prototypen.

Und Prototypen sind nicht schlecht. Aber man sollte den Unterschied kennen.

Agree oder disagree?
```
